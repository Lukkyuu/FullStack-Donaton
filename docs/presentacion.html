Arquitectura de la Plataforma

Arquitectura

BFF & Microservicios Desacoplados

La plataforma humanitaria Donatón se basa en microservicios independientes orquestados por Docker Compose. Un API Gateway (ms-gateway) actúa como BFF centralizando CORS y el enrutamiento. El acoplamiento temporal se elimina mediante mensajería asíncrona distribuida por RabbitMQ.

Cada servicio posee su base de datos aislada, sincronizando estados de forma asíncrona para maximizar la resiliencia y disponibilidad del ecosistema.

Shared Kernel (ms-common)

Para garantizar la integridad en los esquemas de bases de datos compartidas y optimizar el desarrollo, se ha estructurado una biblioteca común: ms-common.

  • Entidades y Modelado JPA: Estructuras de tablas centralizadas.
  • Seguridad y JWT: Filtros e inyección de contexto de seguridad.
  • Mensajería: Estructuras DTO de RabbitMQ consistentes.

Persistencia con JPA y PostgreSQL

Persistencia

Implementación ORM

El mapeo Objeto-Relacional (ORM) se implementa con Spring Data JPA e Hibernate sobre PostgreSQL 15, proveyendo independencia y total portabilidad de datos.

  • HikariCP Connection Pool: Pool de alto rendimiento embebido en Spring Boot para gestionar colas de consultas concurrentes eficientemente.
  • Repository Pattern: interfaces extendidas de JpaRepository que encapsulan las operaciones CRUD y query methods avanzados.
  • Optimización Transaccional: Uso de anotaciones @Transactional en la capa de servicios para asegurar la atomicidad de las operaciones de negocio.

Parámetros del Pool & Dialecto SQL

# application.properties (ms-usuarios)
spring.datasource.url=${DB_URL}
spring.datasource.driver-class-name=org.postgresql.Driver

# Configuración Hikari Pool

spring.datasource.hikari.minimum-idle=5
spring.datasource.hikari.connection-timeout=20000

# Dialecto PostgreSQL e Hibernate
spring.jpa.database-platform=org.hibernate.dialect.PostgreSQLDialect
spring.jpa.hibernate.ddl-auto=update

Modelado y Relaciones de Datos

Modelado

Declaración de Modelos JPA

Las clases modelo Java mapean su lógica estructural a esquemas relacionales a través de anotaciones de Jakarta Persistence:

  • @Entity & @Table: Declara la correspondencia de la clase con una tabla específica de PostgreSQL.
  • @ManyToOne / @JoinColumn: Vincula objetos lógicos (e.g. donaciones y usuarios) creando llaves foráneas automáticas.
  • @Enumerated(EnumType.STRING): Guarda estados inmutables como cadenas en base de datos.
  • @PrePersist: Hook para inyectar marcas temporales en tiempo de ejecución.
@Entity
@Table(name = "donaciones")
public class Donacion {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @ManyToOne
    @JoinColumn(name = "donante_id", nullable = false)
    private Usuario donante;

    @Enumerated(EnumType.STRING)
    private EstadoDonacion estado;

    @Column(name = "fecha_creacion")
    private LocalDateTime fechaCreacion;

    @PrePersist
    protected void onCreate() { fechaCreacion = LocalDateTime.now(); }
}

Procedimientos Almacenados y Triggers

PL/SQL

Optimización de Operaciones Masivas

Para operaciones que exigen alto volumen transaccional o lógica restringida a la base de datos, el backend integra procedimientos almacenados y consultas nativas de PostgreSQL:

  • @Procedure: Permite invocar procedimientos PostgreSQL declarados directamente desde el repositorio JPA, mapeando parámetros automáticos con @Param.
  • @Query(nativeQuery = true): Ejecución directa de consultas SQL optimizadas sobre PostgreSQL, útil para funciones espaciales, de matching, de logs o triggers de base de datos.
// Invocación de Stored Procedure en repositorio Java
public interface DonacionRepository 
       extends JpaRepository<Donacion, Long> {
    
    @Procedure(procedureName = "registrar_donacion_sp")
    Boolean registrarDonacionSp(
        @Param("p_donante_id") Long donanteId,
        @Param("p_descripcion") String descripcion,
        @Param("p_cantidad") Integer cantidad
    );
}

Conclusiones y QA Continuo

Conclusiones

Principales Beneficios

La arquitectura multinivel de pruebas unitarias y de persistencia de Donatón garantiza la estabilidad global y la modularidad del código fuente.

  • Previene Regresiones: Cambios en la biblioteca común ms-common son validados de manera instantánea por Surefire en el reactor Maven.
  • Seguridad de Datos: Mapeo robusto en JPA que encapsula restricciones lógicas a nivel de base de datos sin comprometer la velocidad.

Próximos Pasos de Integración

Para escalar el control de calidad, se proponen las siguientes acciones en el corto plazo:

  • Automatización CI/CD: Incorporar el script de PowerShell en Github Actions para validar pruebas y recrear PDFs en cada pull request.
  • Plugins de Cobertura Backend: Agregar Jacoco al POM parent de Maven para reportes de cobertura nativos en Java.
  • Pruebas E2E: Integrar pruebas funcionales completas con Playwright.
